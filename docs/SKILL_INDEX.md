# Matchy Skill Index

This repo contains project skills for both Claude Code and OpenAI Codex.

## Claude Code

Claude Code reads:

- `CLAUDE.md`
- `.claude/settings.json`
- `.claude/skills/<skill-name>/SKILL.md`

## OpenAI Codex

Codex reads:

- `AGENTS.md`
- `.agents/skills/<skill-name>/SKILL.md`

## Installed skill set

1. `product-truth-compliance` — disclosures, terms, user trust, policy-sensitive flows.
2. `system-architecture` — modules, packages, workers, service boundaries.
3. `database-ledger-design` — schema, event records, ledger integrity, audit records.
4. `yoursafe-payments` — Yoursafe provider integration and external provider events.
5. `entitlement-engine` — plans, pricing rules, account entitlements, state transitions.
6. `moderator-operations` — operator queue, claim/reply, persona card, escalation, QA.
7. `pandora-memory` — memory events, summaries, continuity, risk notes.
8. `email-deliverability` — email jobs, templates, consent, suppression, send limits.
9. `security-privacy-rls` — auth, roles, RLS, private config, privacy lifecycle.
10. `ui-ux-conversion` — user app, operator console, admin console, navigation, screen hierarchy.
11. `qa-acceptance-testing` — validation, E2E checks, acceptance gates.
12. `observability-incident-response` — logs, metrics, alerts, dashboards, runbooks.
13. `growth-retention-ethics` — lifecycle messages, campaign checks, frequency caps.
14. `repo-execution-discipline` — commit hygiene, PR standards, dependency discipline.

## Operating rule

Use these skills before editing their domain. A feature is not complete until its skill acceptance criteria are satisfied or the gap is explicitly reported.
