# UI + UX Conversion

Use before editing product screens.

Rules:
- Do not dump every feature onto one page.
- Every screen must have one primary user and one primary action.
- Show account balance, action cost, and disclosure context where relevant.
- Keep report, block, cancel, and help paths visible where they matter.
- Admin and operator screens must optimize speed, clarity, and low-error execution.

Checks:
- Who is this screen for?
- What decision happens here?
- What must be visible immediately?
- What can move to details or secondary navigation?
- What empty, loading, and error states exist?

Done:
- Screen hierarchy is clear.
- Mobile layout works.
- Critical actions have confirmation.
- Error states are useful.
